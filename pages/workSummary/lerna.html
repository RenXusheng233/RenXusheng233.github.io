# Defination

# Lerna

Lerna is a tool that optimizes the workflow around managing multi-package repositories with git and npm.

# monorepo

In version control systems, a monorepo ("mono" meaning 'single' and "repo" being short for 'repository') is a software development strategy where code for many projects is stored in the same repository. - Wikipedia

# Getting Started

# Commands

# This flag tells Lerna to use independent versioning mode.
$ lerna init -i

# Bootstrap the packages in the current Lerna repo. 
$ lerna bootstrap

# Import the package at <path-to-external-repository>, with commit history, into packages/<directory-name>
# Original commit authors, dates and messages are preserved.
$ lerna import <pathToRepo>

# --scope
# --dev
# --exact
# --peer
$ lerna add <package>[@version]

# create a package
$ lerna create <package>

# --dist-tag [tagname]
# --canary/-c
# --skip-git
# --force-publish [packages]
$ lerna publish

# Check which packages have changed since the last release.
$ lerna changed

# Diff all packages or a single package since the last release.
$ lerna diff [package?]

#Run an npm script in each package that contains that script.
$ lerna run [script]

# Run a command in each package.
$ lerna exec -- <command> [...args]

# List all of the public packages in the current Lerna repo.
$ lerna ls

# Remove the node_modules directory from all packages.
$ lerna clean

# Config

// lerna.json
{
    "useWorkspaces": true,
    "version": "independent",
    "npmClient": "yarn",
}

// package.json
{
    "private": true,
    "workspaces": [
        "packages/*"
    ],
}
